KEY STRUCTURAL NOTES (your setup vs original build)
================================================================

1. checklists/ is a standalone module
   checklist.controller, service, and validator live here.
   There is no checklist.routes.ts — the routes are registered
   directly in engagements/engagement.routes.ts which imports
   checklistController from '../checklists/checklist.controller.js'

2. questionnaires/ is a standalone module
   All 5 questionnaire files live here including the model.
   questionnaire.routes.ts exports TWO routers:
     - engagementQuestionnaireRouter → imported by engagement.routes.ts
       for nested /engagements/:id/questionnaires paths
     - questionnaireRouter → imported by app.ts
       for standalone /questionnaires/:id/... paths

3. feedbacks/ folder name is plural
   All imports in app.ts and within the module use this exact name.
   No mismatch — consistent throughout.

4. engagements/ no longer contains questionnaire or checklist files
   Those moved to their own modules. engagements/ now only contains
   the core engagement files + purchaseQuestionnaire.model.ts
   (purchase questionnaire stays in engagements because it is
   created exclusively by the purchase flow, not by the admin)
