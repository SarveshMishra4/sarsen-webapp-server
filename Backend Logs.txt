    4. Incremental Roadmap (Very Important)

    We will build the backend in 8 layers.
    LAYER 1 — Public Data Collection (Already Done)
        Capabilities:
            Newsletter subscription
            Contact form submission
        Modules:
            newsletter
            contact
        Goal achieved:
            Users can interact with your website.

    LAYER 2 — Admin System
    Next capability:
        System owner can control the platform.
    New module:
        modules/admin
    Capabilities inside it:
        Create admin
        Login admin
        Verify admin identity
        Protect admin routes
    Behavior:
        Admin logs in
            ↓
        Server generates token
            ↓
        Token required to access admin APIs
    Result:
        Private control over the system.

    LAYER 3 — Admin Dashboard Data
    Now admin needs to **see data**.
    Capabilities:
    View newsletter subscribers
    Delete subscriber
    View contacts
    Mark contact resolved
    Delete contact
    ```

    Modules affected:

    ```
    newsletter
    contact
    ```

    But we will **not modify them directly**.

    Instead we add **admin routes**.

    Example:

    ```
    /admin/newsletter
    /admin/contact
    ```

    Result:

    ```
    Admin can manage collected data.
    ```
